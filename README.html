PHASE 1 サーバー起動時のエラー
Q 01 サーバーが起動しない settings.py 初級 ▶
Terminal Error
django.core.exceptions.ImproperlyConfigured: 'posts' is not installed. Add 'posts' to INSTALLED_APPS.
タスク： ターミナルのエラーを読んで、settings.py を修正してください。
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'django_extensions',
    # ← 何かが足りない
]
🔑 解答を見る

原因： posts アプリが INSTALLED_APPS に登録されておらず、Django がアプリの存在を知りません。

✔ 修正
    'django_extensions',
    'posts',  # ← 追加
]

💡 ポイント： startapp したら INSTALLED_APPS 追加 → makemigrations → migrate の3セットが基本です。

Q 02 まだサーバーが起動しない config/urls.py 初級 ▶
Terminal Error
ModuleNotFoundError: No module named 'urls'
タスク： include() に渡すパスが間違っています。正しいモジュールパスに直してください。
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('posts/', include('urls')),  # ← ここがおかしい
]
🔑 解答を見る

原因： 'urls' というモジュールは存在しません。アプリ名を含めたドット記法で指定します。

✔ 修正
path('posts/', include('posts.urls')),

💡 ポイント： include() には 'アプリ名.urls' の形式で渡します。

PHASE 2 ページ表示時のエラー
Q 03 ページを開くと NoReverseMatch posts/urls.py 初級 ▶
Browser Error
NoReverseMatch: 'posts' is not a registered namespace
タスク： 名前空間の設定を確認して修正してください。
from django.urls import path
from . import views

app_name = 'post'  # ← views.py では 'posts:index' と書いている

urlpatterns = [
    path('', views.IndexView.as_view(), name='index'),
    path('create/', views.CreateView.as_view(), name='create'),
]
🔑 解答を見る

原因： app_name = 'post' は 's' が抜けたタイポです。redirect('posts:index') や {% url 'posts:create' %} と一致しません。

✔ 修正
app_name = 'posts'

💡 ポイント： app_name は redirect() や {% url %} の名前空間と完全一致が必要です。1文字でも違うと即エラーになります。

Q 04 マイグレーションが通らない models.py 初級 ▶
Terminal Error (makemigrations)
SystemCheckError: System check identified 1 issue: posts.Post.content: (fields.E120) CharFields must define a 'max_length' attribute.
タスク： models.py を修正して makemigrations → migrate を通してください。
from django.db import models
from django.utils import timezone

class Post(models.Model):
    class Meta:
        db_table = 'posts'

    content = models.CharField(blank=True)  # ← 必須の引数が抜けている
    created_at = models.DateTimeField(default=timezone.now)
🔑 解答を見る

原因： CharField には max_length が必須です。省略するとシステムチェックで弾かれ、マイグレーションが作成されません。

✔ 修正
content = models.CharField(max_length=255, blank=True)

💡 ポイント： 修正後は python manage.py makemigrations → python manage.py migrate の実行を忘れずに。

Q 05 ページを開くと TemplateDoesNotExist views.py 初級 ▶
Browser Error
TemplateDoesNotExist: index.html
タスク： テンプレートのパス指定を確認して修正してください。テンプレートファイルは templates/posts/index.html にあります。
class IndexView(View):
    def get(self, request, *args, **kwargs):
        posts = Post.objects.all()
        return render(request, 'index.html', {'posts': posts})  # ← パスが違う
🔑 解答を見る

原因： テンプレートファイルは templates/posts/index.html に置かれているのに 'index.html' と指定しているため見つかりません。

✔ 修正
return render(request, 'posts/index.html', {'posts': posts})

💡 ポイント： アプリ間の名前衝突を防ぐため、テンプレートは templates/アプリ名/ 以下に置くのが慣習です。

Q 06 一覧ページで NoReverseMatch index.html 初級 ▶
Browser Error
NoReverseMatch: Reverse for 'create' not found. 'create' is not a valid view function or pattern name.
タスク： {% url %} タグの記述を確認して修正してください。
<a href="{% url 'create' %}">新規投稿</a>  {# ← 名前空間が抜けている #}
🔑 解答を見る

原因： app_name = 'posts' を定義しているので、テンプレートの URL 逆引きも名前空間付きにする必要があります。

✔ 修正
<a href="{% url 'posts:create' %}">新規投稿</a>

💡 ポイント： {% url %} と redirect() は同じルール。app_name を設定したらすべての逆引きに 名前空間: を付けます。

PHASE 3 フォーム送信・DB操作のエラー
Q 07 投稿ボタンを押すと 403 Forbidden create.html 初級 ▶
Browser Error
403 Forbidden CSRF verification failed. Request aborted.
タスク： フォームのテンプレートに必要なものが抜けています。追加してください。
<form method="post">
    {# ← 何かが足りない #}
    {{ form.content }}
    <button type="submit">投稿する</button>
</form>
🔑 解答を見る

原因： Django は POST リクエストに CSRF トークンを要求します。これがないと不正リクエストとみなして 403 を返します。

✔ 修正
<form method="post">
  {% csrf_token %}
  {{ form.content }}
  <button type="submit">投稿する</button>
</form>

💡 ポイント： method="post" のフォームには 必ず {% csrf_token %} を書きます。CSRF（クロスサイトリクエストフォージェリ）攻撃を防ぐ仕組みです。

Q 08 投稿ボタンを押すと TypeError forms.py 初級 ▶
Browser Error
TypeError: PostForm.Meta.fields cannot be a string. Did you mean to type: fields = ['content']?
タスク： fields の書き方を修正してください。
class PostForm(forms.ModelForm):
    class Meta:
        model = Post
        fields = 'content'  # ← 型が間違っている
🔑 解答を見る

原因： fields はリストで渡す必要があります。文字列で渡すと各文字（'c','o','n'...）をフィールド名として解釈しようとして TypeError になります。

✔ 修正
fields = ['content']

💡 ポイント： 全フィールドを対象にする場合は fields = '__all__'（文字列）が使えますが、それ以外はリストで明示するのがベストプラクティスです。

Q 09 投稿後に NoReverseMatch views.py 初級 ▶
Browser Error
NoReverseMatch: Reverse for 'index' not found. 'index' is not a valid view function or pattern name.
タスク： redirect() の引数を確認して修正してください。
def post(self, request, *args, **kwargs):
    form = PostForm(request.POST)
    if form.is_valid():
        form.save()
    return redirect('index')  # ← Q06 と同じ問題が views.py にもある
🔑 解答を見る

原因： app_name = 'posts' を定義しているので redirect() も名前空間付きにする必要があります。

✔ 修正
return redirect('posts:index')

💡 ポイント： {% url %}（テンプレート）と redirect()（ビュー）は同じ逆引き機能です。app_name を設定した場合、両方に 名前空間: が必要です。

Q 10 空のまま投稿できてしまう（エラーが表示されない） views.py 中級 ▶
動作の異常（エラーは出ないが正しく動いていない）
空のフォームを送信してもエラーが表示されず、 何も入力していないのに一覧ページへ戻ってしまう。
タスク： バリデーション失敗時の処理が抜けています。エラーをフォームに表示するよう修正してください。
def post(self, request, *args, **kwargs):
    form = PostForm(request.POST)
    if form.is_valid():
        form.save()
    return redirect('posts:index')  # ← is_valid() が False でもここに来てしまう
🔑 解答を見る

原因： if form.is_valid() が False（バリデーション失敗）でも return redirect() に到達してしまい、エラーメッセージが表示されません。

✔ 修正
def post(self, request, *args, **kwargs):
    form = PostForm(request.POST)
    if form.is_valid():
        form.save()
        return redirect('posts:index')  # 成功時だけ redirect
    return render(request, 'posts/create.html', {'form': form})  # 失敗時は再描画

💡 ポイント： POST処理の基本パターン

✅ 成功時 → redirect()（リロードによる二重送信を防ぐ）

❌ 失敗時 → render()（エラーメッセージ付きフォームを再表示）
