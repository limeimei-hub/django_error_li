Q 01 サーバーが起動しない config_urls.py 初級 ▶
Terminal Error
ModuleNotFoundError: No module named 'urls'
タスク： include() に渡すパスを正しく修正してください。
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('posts/', include('urls')),  # ← ここがおかしい
]
🔑 解答を見る

原因： 'urls' というモジュールは存在しません。アプリ名を含めたドット記法で指定します。

✔ 修正後
path('posts/', include('posts.urls')),

💡 include() には 'アプリ名.urls' の形式で渡します。

Q 02 /posts/ を開くと NoReverseMatch posts_urls.py 初級 ▶
⚠ このファイルには エラーが2つ あります。今回は1つ目だけ直してください。2つ目は Q06 で直します。
Browser Error
NoReverseMatch: 'posts' is not a registered namespace
タスク： 名前空間の設定を確認して修正してください。
from django.urls import path
from . import views

app_name = 'post'  # ← 今回直す箇所

urlpatterns = [
    path('', views.IndexView.as_view(), name='index'),
    path('creates/', views.CreateView.as_view(), name='create'),  # ← Q06 で直す
]
🔑 解答を見る

原因： 'post' は 's' が抜けたタイポです。redirect('posts:index') などと一致しません。

✔ 修正後（今回直す箇所のみ）
app_name = 'posts'

⚠ 'creates/' の行は今回触らないでください。Q06 で直します。

Q 03 /posts/ を開くと TemplateDoesNotExist views.py 初級 ▶
⚠ このファイルには エラーが3つ あります。今回は1つ目だけ直してください。残りは Q09・Q10 で直します。
Browser Error
TemplateDoesNotExist: index.html
タスク： テンプレートのパス指定を修正してください。テンプレートファイルは templates/posts/index.html にあります。
class IndexView(View):
    def get(self, request, *args, **kwargs):
        posts = Post.objects.all()
        return render(request, 'index.html', {'posts': posts})  # ← 今回直す箇所

class CreateView(View):
    def post(self, request, *args, **kwargs):
        form = PostForm(request.POST)
        if form.is_valid():
            form.save  # ← Q10 で直す
            return redirect('index')  # ← Q09 で直す
        return render(request, 'posts/create.html', {'form': form})
🔑 解答を見る

原因： テンプレートは templates/posts/index.html にありますが 'index.html' と指定しているため見つかりません。

✔ 修正後（今回直す箇所のみ）
return render(request, 'posts/index.html', {'posts': posts})

⚠ form.save と redirect('index') は今回触らないでください。Q09・Q10 で直します。

Q 04 ページは表示されるが CSS が当たっていない settings.py 確認 ▶
動作異常（エラーは出ない）
ページは表示されるが CSS が反映されず見た目が崩れている。 ブラウザの開発者ツール（F12）→ ネットワークタブで CSS が 404 になっている。
タスク： 静的ファイルの設定を確認して修正してください。静的ファイルのディレクトリ名は static です。
STATIC_URL = '/static/'
STATICFILES_DIRS = [BASE_DIR / 'statics']  # ← ここがおかしい
🔑 解答を見る

原因： 実際のディレクトリ名は static ですが 'statics' と指定しているため CSS が見つかりません。

✔ 修正後
STATICFILES_DIRS = [BASE_DIR / 'static']

💡 CSS が表示されないときはブラウザの開発者ツール（F12）のネットワークタブで 404 になっていないか確認しましょう。

Q 05 「新規投稿」リンクで NoReverseMatch index.html 初級 ▶
Browser Error
NoReverseMatch: Reverse for 'create' not found. 'create' is not a valid view function or pattern name.
タスク： {% url %} タグの記述を修正してください。
<a href="{% url 'create' %}">新規投稿</a>  {# ← 名前空間が抜けている #}
🔑 解答を見る

原因： app_name = 'posts' を定義しているため {% url %} も名前空間付きにする必要があります。

✔ 修正後
<a href="{% url 'posts:create' %}">新規投稿</a>

💡 {% url %} と redirect() は同じルール。app_name を設定したら両方に 名前空間: が必要です。

Q 06 投稿フォームを開くと 404 posts_urls.py 初級 ▶
⚠ このファイルの Q02 は修正済みのはずです。今回は2つ目のエラーを直してください。
Browser Error
Page not found (404) /posts/create/ に一致するURLパターンが見つかりませんでした
タスク： URL パターンの定義を確認して修正してください。
urlpatterns = [
    path('', views.IndexView.as_view(), name='index'),
    path('creates/', views.CreateView.as_view(), name='create'),  # ← 今回直す箇所
]
🔑 解答を見る

原因： 'creates/' と定義されているため /posts/create/ へのアクセスが 404 になります。

✔ 修正後
path('create/', views.CreateView.as_view(), name='create'),

💡 404 が出たときはまず urls.py のパス定義を見直しましょう。

Q 07 投稿ボタンを押すと 403 Forbidden create.html 初級 ▶
Browser Error
403 Forbidden CSRF verification failed. Request aborted.
タスク： フォームに必要なものが抜けています。追加してください。
<form method="post">
  {# ← 何かが足りない #}
  {{ form.content }}
  <button type="submit">投稿する</button>
</form>
🔑 解答を見る

原因： Django は POST リクエストに CSRF トークンを要求します。これがないと 403 を返します。

✔ 修正後
<form method="post">
  {% csrf_token %}
  {{ form.content }}
  <button type="submit">投稿する</button>
</form>

💡 method="post" のフォームには必ず {% csrf_token %} を書きます。

Q 08 投稿ボタンを押すと NameError forms.py 初級 ▶
Browser Error
NameError: name 'Posts' is not defined
タスク： forms.py のモデル指定を確認して修正してください。
class PostForm(forms.ModelForm):
    class Meta:
        model = Posts  # ← モデル名が間違っている
        fields = ['content']
🔑 解答を見る

原因： モデル名が Posts（s が余分）になっています。定義されていないクラス名を参照しているため NameError になります。

✔ 修正後
model = Post

💡 ポイント： NameError: name 'XX' is not defined は「そのクラスや変数が存在しない」というエラーです。スペルミスがないか確認しましょう。

Q 09 投稿後に NoReverseMatch views.py 初級 ▶
⚠ このファイルの Q03 は修正済みのはずです。今回は2つ目のエラーを直してください。3つ目は Q10 で直します。
Browser Error
NoReverseMatch: Reverse for 'index' not found. 'index' is not a valid view function or pattern name.
タスク： redirect() の引数を修正してください。
if form.is_valid():
    form.save  # ← Q10 で直す
    return redirect('index')  # ← 今回直す箇所
🔑 解答を見る

原因： app_name = 'posts' を定義しているため redirect() も名前空間付きにする必要があります。

✔ 修正後（今回直す箇所のみ）
return redirect('posts:index')

⚠ form.save は今回触らないでください。Q10 で直します。

Q 10 投稿しても一覧に表示されない views.py 確認 ▶
⚠ このファイルの Q03・Q09 は修正済みのはずです。今回は最後のエラーを直してください。
動作異常（エラーは出ない）
投稿ボタンを押しても一覧ページに投稿が表示されない。 エラーメッセージは何も出ない。
タスク： 投稿が保存されない原因を見つけて修正してください。
if form.is_valid():
    form.save  # ← ここがおかしい
    return redirect('posts:index')
🔑 解答を見る

原因： form.save は括弧がないためメソッドを呼び出していません。DB への保存が行われないまま処理が進みます。Python はこれをエラーにしないため気づきにくいバグです。

✔ 修正後
form.save()  # () を付けて呼び出す

💡 「エラーは出ないのに動かない」場合は () 抜けを疑いましょう。form.save・post.save など括弧なしでは何も実行されません。
